# 🍲 myRecipes – Gerenciador de Receitas
Este projeto é um **CRUD completo de receitas com autenticação de usuários**.  
Cada usuário pode criar sua conta, fazer login e gerenciar suas próprias receitas.  

O objetivo principal foi **aprofundar os conhecimentos em desenvolvimento full-stack**, integrando **frontend e backend** com autenticação JWT, upload de imagens com Cloudinary e manipulação de dados com MongoDB.

# 🚀 Funcionalidades

### 🔐 Autenticação
- Registro de contas
- Login de usuários
- Tokens JWT para autenticação

### 📖 CRUD de Receitas
- Criar receitas (com imagens)
- Listar receitas
- Atualizar receitas
- Deletar receitas

### 💬 CRUD de Comentarios
- Criar comentarios
- Listar comentarios
- Atualizar comentarios
- Deletar comentarios

### ❤️ Interações
- Curtir receitas
- Comentar receitas
- Sistema de categorias
- Pesquisa de receitas

# 🧑‍💻 Tecnologias utilizadas

### 🌐 Front-end
- **TypeScript** (tipagem estática)
- **React** (biblioteca para UI)
- **React-router** (gerenciamento de rotas)
- **tailwindcss** (estilização utilitária)
- **HTML5 & CSS3**

### 🔙 Back-end
- **TypeScript**
- **Express** (Framework Web)
- **MongoDB** (Banco NoSQL)
- **Mongoose** (ODM)
- **JWT** (autenticação segura)
- **bcrypt** (hash de senhas para segurança)
- **Cloudinary** (armazenamento de imagens)

# 📦 Instalação local

### 1. Clone o repositório

```
git clone https://github.com/Leleo1337/myrecipes.git
```

### 2. Instale as dependências
(necessário ter Node.js instalado)  
```
npm install
```

### 3. Configure o ambiente
Crie um arquivo `.env` na raiz do projeto:  

```
VITE_API_URL=https://recipes-backend-beta.vercel.app
VITE_CLOUD_NAME=NOMEDASUANUVEMDOCLOUDINARY
```

### 4. Execute o projeto
```
npm run dev
```
